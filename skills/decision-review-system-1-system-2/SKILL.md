# 管理決策的 System 1／System 2 偏誤檢核

## 適用範圍

管理者在重大人事、專案、資源分配、醫療／研究管理或策略選擇前的個人決策審查。

## 證據基礎

本 skill 的所有概念均可在 `templates/source-claim-ledger.md` 中找到對應的 claim ID（C-001 ~ C-086）。每個 claim 均附有 B4（原文）、B4.1（30分鐘摘要）、B4.2（中譯本）的逐字引用與位置。所有 claim 的 `status` 目前為 `draft`，尚未經人工確認。

## 互動協定

1. 一次只問一個澄清問題。
2. 先進入 Divergent phase，允許產生 `idea` 與 `hypothesis`。
3. 再進入 Convergent phase，檢查事實、base rate、反證、替代方案與風險。
4. 資料不足時標記 `insufficient evidence`，不擅自補完。
5. 最後產出 `templates/decision-review.md` 格式的報告。

## Phase 0：決策問題界定

在進入 Divergent phase 之前，先確認決策的基本框架。

| # | 問題 | 目的 | 對應 claim |
|---|---|---|---|
| 0.1 | 你正在考慮的決策是什麼？請用一句話描述。 | 界定決策本身 | — |
| 0.2 | 這個決策最晚何時需要做出？ | 確認時間壓力（時間壓力有利 System 1，不利 System 2） | C-002 |
| 0.3 | 這個決策的範圍有多大？涉及哪些人、多少資源？ | 評估賭注大小（賭注高時應更努力避免錯誤） | C-082 |
| 0.4 | 這個決策可逆嗎？如果做錯了，能回頭嗎？ | 評估可逆性（不可逆決策需更謹慎） | — |

## Phase 1：Divergent — 直覺與假設展開

此階段允許自由聯想，產生 `idea` 與 `hypothesis`，不急於評判。目的是把 System 1 的直覺反應外顯化，方便後續檢查。

### 1A. 直覺外顯

| # | 問題 | 目的 | 對應 claim |
|---|---|---|---|
| 1.1 | 你對這個決策的第一直覺是什麼？傾向 go 還是 stop？ | 外顯 System 1 直覺 | C-001, C-002 |
| 1.2 | 這個直覺背後的故事是什麼？你腦中浮現了哪些畫面或案例？ | 檢查 WYSIATI — System 1 用手頭資訊建構的故事 | C-005, C-006 |
| 1.3 | 你對這個直覺有多大的信心？（1-10 分） | 記錄主觀信心基線（注意：信心不等於準確性） | C-006, C-023, C-069 |
| 1.4 | 這個直覺讓你聯想到什麼情緒？（興奮、恐懼、焦慮、安心？） | 記錄情緒信號（情緒可能正在引導判斷） | C-009, C-010 |

### 1B. 假設與假說列舉

| # | 問題 | 目的 | 對應 claim |
|---|---|---|---|
| 1.5 | 這個決策背後有哪些你還沒驗證的假設？ | 列舉隱含假設 | C-005 |
| 1.6 | 如果這個決策成功了，你認為主要原因會是什麼？ | 檢查敘事謬誤 — 成功故事常過度歸因才能而輕視運氣 | C-031 |
| 1.7 | 是什麼讓你覺得這個方案可行？第一次聽到或想到它是什麼情境？ | 檢查可得性 — 方案可能只是因為容易想到 | C-015, C-016 |
| 1.8 | 有沒有一個數字或參考點一直盤據在你腦中？（例如預算上限、他人報價、歷史數據） | 檢查錨定 — 先前接觸的數字可能正在影響估計 | C-011, C-012 |
| 1.9 | 這個決策是怎麼被「框」起來的？如果換一個問法，你還會做同樣的選擇嗎？ | 檢查框架效應 | C-048, C-050 |

## Phase 2：Convergent — 證據與偏誤檢核

此階段進入 System 2 分析，逐一檢查證據品質、base rate、反證、替代方案與風險。

### 2A. 證據檢核

| # | 問題 | 目的 | 對應 claim |
|---|---|---|---|
| 2.1 | 你目前掌握了哪些「事實」？這些事實的來源是什麼？ | 區分 fact 與 assumption | C-005, C-008 |
| 2.2 | 你可能缺少哪些資訊？有哪些你知道自己不知道的？ | 補強 WYSIATI 盲區 | C-005, C-008 |
| 2.3 | 這個決策是基於單一來源（一份報告、一個人的意見）嗎？ | 檢查資訊來源多樣性 | C-008, C-021, C-022 |
| 2.4 | 如果要你說服一個反對者，你會用什麼證據？ | 測試證據強度 | — |

### 2B. Base Rate / 外部視角

| # | 問題 | 目的 | 對應 claim |
|---|---|---|---|
| 2.5 | 在類似的情況下，別人的成功率是多少？（參考類別的基礎比率） | 引入外部視角 | C-032, C-034, C-036, C-038 |
| 2.6 | 我們如何知道這一次跟參考類別不同？有什麼具體證據支持「這次不一樣」？ | 檢查內部意見偏見 | C-035, C-036 |
| 2.7 | 這個專案/計劃的時間和成本預估，是否參考了同類專案的分佈？還是只看最佳情境？ | 檢查規劃謬誤 | C-037, C-039, C-040 |
| 2.8 | 如果證據薄弱，你是否應該回到基礎比率？ | 證據薄弱時堅持 base rate | C-034 |

### 2C. 偏誤逐項檢查

| # | 問題 | 目的 | 對應 claim |
|---|---|---|---|
| 2.9 | **錨定**：有沒有一個數字不斷影響你的估計？你是否嘗試過從零開始重新估算？ | 檢查錨定效應 | C-011, C-013 |
| 2.10 | **可得性**：你想到的案例是因為它真的常見，還是因為它特別生動/最近發生？ | 檢查可得性偏見 | C-015, C-017 |
| 2.11 | **框架**：如果把「獲利」改說成「避免損失」，或把「風險」改說成「機會」，你的決策會改變嗎？ | 檢查框架效應 | C-048, C-049 |
| 2.12 | **過度自信**：你對預測的信心是否高於應有水準？你有沒有考慮過自己可能錯在哪裡？ | 檢查過度自信 | C-023, C-025, C-026 |
| 2.13 | **確認偏誤**：你是否主動尋找過反證？還是只在找支持自己方案的資料？ | 檢查確認偏誤 | C-018, C-019 |
| 2.14 | **後見之明/結果偏誤**：如果這個決策未來失敗了，你預期別人會怎麼評價你？這是否正在影響你現在的選擇？ | 檢查後見之明壓力 | C-027, C-028, C-029 |
| 2.15 | **損失厭惡**：你是否因為害怕失去現有資源/地位而拒絕一個值得冒險的方案？ | 檢查損失厭惡 | C-052, C-055, C-066 |
| 2.16 | **沉沒成本**：你是否因為已經投入了時間/金錢而難以放棄？ | 檢查沉沒成本謬誤 | C-059, C-060 |
| 2.17 | **月暈效應**：你是否因為對某個人的整體印象而過度信任或否定他的提案？ | 檢查月暈效應 | C-020 |

### 2D. 替代方案

| # | 問題 | 目的 | 對應 claim |
|---|---|---|---|
| 2.18 | 除了目前的方案，還有哪些替代方案？至少列出 3 個。 | 強制考慮替代方案 | C-050 |
| 2.19 | 「不做事」也是一個選項嗎？維持現狀的代價是什麼？ | 檢查不作為偏誤 | — |
| 2.20 | 是否可以把大決策拆成幾個小決策，分階段執行？ | 引入試點/階段性決策 | C-076, C-082 |

### 2E. 事前驗屍

| # | 問題 | 目的 | 對應 claim |
|---|---|---|---|
| 2.21 | 假設一年後這個決策被執行了，但結果慘敗。請用 5-10 分鐘寫下慘敗的可能原因。 | 事前驗屍 | C-045, C-046, C-047 |
| 2.22 | 在這些失敗原因中，哪些是你現在就能預防的？ | 將事前驗屍轉化為行動 | C-047 |

## Phase 3：Pilot Design — 試點設計

如果決策傾向 go，但在 Convergent phase 仍有不確定性，建議設計小規模試點。

| # | 問題 | 目的 | 對應 claim |
|---|---|---|---|
| 3.1 | 這個方案的最小可測試版本是什麼？ | 縮小範圍 | C-076, C-082 |
| 3.2 | **Outcome metrics**：試點成功時，你會看到什麼具體結果？ | 定義結果指標 | — |
| 3.3 | **Process metrics**：試點過程中，你要追蹤哪些過程指標？ | 定義過程指標 | — |
| 3.4 | **Guardrail metrics**：哪些指標一旦觸發就必須停止？ | 定義護欄指標 | — |
| 3.5 | **go threshold**：達到什麼條件才擴大實施？ | 定義 go 門檻 | — |
| 3.6 | **revise threshold**：達到什麼條件要修改方案後再試？ | 定義 revise 門檻 | — |
| 3.7 | **stop threshold**：達到什麼條件必須完全停止？ | 定義 stop 門檻 | — |
| 3.8 | 試點的檢討日期是什麼時候？ | 預設檢討時點 | — |

## Phase 4：報告產出

完成所有問題後，依 `templates/decision-review.md` 格式產出結構化決策審查報告。

報告必須包含：
1. 決策問題（Phase 0）
2. 證據與假設（Phase 2A）
3. System 1 記錄（Phase 1A）
4. System 2 分析（Phase 2B-2C）
5. 偏誤檢查結果（Phase 2C，逐項）
6. 試點設計（Phase 3，如適用）
7. 決策建議與信心水準

每項結論須標記 `fact` / `hypothesis` / `idea` / `inference` / `insufficient evidence`。

## 案例層

可引用 `case-library/` 的三國案例作為類比與教學示範，但必須標記小說、史料、現代研究或 `inference`，不得將類比當成因果證據。

## 限制

- 本 skill 不取代臨床判斷、倫理審查、統計分析或正式治理程序。
- 所有 claim 引用目前 status 為 `draft`，人工確認前不得用於 `reviewed` 以上之決策。
- 本 skill 是決策品質檢核工具，不是決策替代品。最終決策權屬於管理者。
